# [Spring] Application Context 란?
## 중요도 ★★★
### 정의
* 애플리케이션의 빈 객체들을 생성, 관리, 제어하는 역할
* 빈: spring에서 관리되는 객체, IOC 컨테이너에 의해 생성되고 의존성 주입(DI)을 받을 수 있음
* xml, java설정 클래스, 어노테이션을 통해 빈의 정의와 의존성 설정 간으

### 역할
* 빈 객체의 생성과 소멸 담당, 객체의 생명 주기 제어, 객체의 초기화와 종료를 담당하여 애플리케이션의 안정성과 효율성을 증진
* 의존성주입(DI) 관리 측면 - 빈 객체 간의 의존성을 주입하여 객체간의 결합도를 낮춤, 코드의 재사용성과 유연성 향상
* properties 설정 및 환경 관리 측면 - 설정파일이나 환경변수를 통해 애플리케이션 동작 관리, 다양한 환경에서 실행가능하게 함
* 리소스 관리 측면 - 애플리케이션의 파일, 데이터베이스(실무에선 context-datasource.xml로 따로 관리) 등을 관리하고 접근할 수 있게 함, 리소스에 대한 추상화 계층을 제공하여 편리한 리소스 관리를 가능케 함
  
### 사용법
* XML 설정 - context-application.xml 과 같은 설정 파일을 작성
* Java 설정 클래스 - @Configuration 어노테이션을 사용하여 Java 클래스를 설정 파일로 활용가능, @Bean 어노테이션을 이용하여 빈의 정의 작성, @Autowired를 사용하여 의존성 주입 설정 가능
* 어노테이션 - @ComponentScan 어노테이션을 사용하여 자동으로 빈 스캔 및 등록 가능, @Autowired를 사용하여 의존성 주입 설정 가능
